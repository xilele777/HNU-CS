# 二、程序填空题（10 分，每空 2 分）

如下是一个 C 语言程序及其对应的汇编代码（32 位机，小端环境下编译），请参照汇编代码，完成 C 程序的空缺部分。

## C 语言程序

```c
int main()
{
    int i, j, flag;
    int sum = (1)____________;
    for ((2)____________; (3)____________; i++, j += 2)
    {
        flag = (4)____________;
        sum += (5)____________;
    }
    return sum;
}
```

## 汇编代码

```asm
main:
    pushl   %ebp
    movl    %esp, %ebp
    subl    $16, %esp
    movl    $17, -8(%ebp)
    movl    $5, -16(%ebp)
    movl    $1, -12(%ebp)
    jmp     .L2
.L5:
    movl    -16(%ebp), %eax
    andl    $1, %eax
    testl   %eax, %eax
    jne     .L3
    movl    -12(%ebp), %eax
    addl    $3, %eax
    jmp     .L4
.L3:
    movl    -16(%ebp), %eax
    subl    $2, %eax
.L4:
    movl    %eax, -4(%ebp)
    subl    $1, -16(%ebp)
    movl    -12(%ebp), %eax
    imull   -16(%ebp), %eax
    addl    -4(%ebp), %eax
    addl    %eax, -8(%ebp)
    addl    $1, -12(%ebp)
    addl    $1, -16(%ebp)
    addl    $2, -12(%ebp)
.L2:
    movl    -12(%ebp), %eax
    movl    -16(%ebp), %edx
    addl    %edx, %eax
    cmpl    $98, %eax
    jle     .L5
    movl    -8(%ebp), %eax
    leave
    ret
```

---

## 答案

```c
int main()
{
    int i, j, flag;
    int sum = 17;                                          // (1)
    for (i = 5, j = 1; i + j <= 98; i++, j += 2)          // (2) (3)
    {
        flag = (i % 2 == 0) ? (j + 3) : (i - 2);          // (4)
        sum += j++ * (i - 1) + flag;                       // (5)
    }
    return sum;
}
```

| 空号 | 答案 |
|:----:|:-----|
| (1) | `17` |
| (2) | `i = 5, j = 1` |
| (3) | `i + j <= 98` |
| (4) | `i % 2 == 0 ? j + 3 : i - 2` |
| (5) | `j++ * (i - 1) + flag` |

---

## 解析

### 栈帧布局

| 偏移量 | 变量 | 初值 |
|:------:|:----:|:----:|
| `-4(%ebp)` | flag | — |
| `-8(%ebp)` | sum | 17 |
| `-12(%ebp)` | **j** | 1 |
| `-16(%ebp)` | **i** | 5 |

### (1) 由 `movl $17, -8(%ebp)` 知 sum 初始化为 `17`

### (2) 由 `movl $5, -16(%ebp)` 和 `movl $1, -12(%ebp)` 知初始化 `i = 5, j = 1`

### (3) 标号 `.L2` 处：`movl -12(%ebp), %eax; movl -16(%ebp), %edx; addl %edx, %eax` 计算 `j + i`，`cmpl $98, %eax` 后 `jle .L5`，即条件为 `i + j <= 98`

### (4) 标号 `.L5` 处：`movl -16(%ebp), %eax`（取 i），`andl $1, %eax` 测试 i 奇偶性。偶数分支取 `j+3`，奇数分支取 `i-2`

### (5) `subl $1, -16(%ebp)` 是表达式中的 `i-1`（编译器将 i 暂减 1 来参与计算）。`movl -12(%ebp), %eax` 先读取 j 旧值，`imull` 计算 `j × (i-1)`，再加 flag 累加到 sum。随后 `addl $1, -12(%ebp)` 是表达式里 `j++` 的后置自增——先用旧值乘，再自增

---

## 变量与栈偏移的对应关系如何确定？

这是本题最关键的一步。汇编代码中只出现 `-4(%ebp)`、`-8(%ebp)`、`-12(%ebp)`、`-16(%ebp)`，需要推断每个偏移对应哪个 C 变量。方法如下：

### 第一步：定位 sum（`-8(%ebp)`）

汇编中 `movl $17, -8(%ebp)` 是唯一一个用**立即数**初始化的变量。C 程序中只有 `int sum = (1)____` 需要一个初始值。因此：

> `-8(%ebp)` = **sum**，且 (1) = 17

这一步是**通过语义匹配**确定的——C 代码里哪个变量需要给初值，汇编里哪个偏移就被赋了立即数。

### 第二步：圈定 i 和 j 的范围（`-12` 和 `-16`）

剩下三个偏移：`-4(%ebp)`、`-12(%ebp)`、`-16(%ebp)`，对应 i、j、flag。

看循环条件（标号 `.L2`）：

```asm
movl -12(%ebp), %eax   ; 加载 -12
movl -16(%ebp), %edx   ; 加载 -16
addl %edx, %eax        ; 相加
cmpl $98, %eax         ; 与 98 比较
```

for 的条件是 (3)，只能涉及 i 和 j。其中 `-12` 和 `-16` 的值被相加后和 98 比较，所以：

> **`-12(%ebp)` 和 `-16(%ebp)` 是 i 和 j**（具体谁是 i 谁是 j 见下一步）

剩余的 `-4(%ebp)` 就是 **flag**。

### 第三步：区分 i 与 j（谁是谁？）

这是最关键的一步。我们需要从汇编中找出哪个偏移是 i、哪个是 j。

#### 真正的决定性线索：for 增量中的 `+2`

C 代码的 for 增量是 **`i++, j+=2`**，即：
- i 每轮 **+1**
- j 每轮 **+2**

现在看汇编中循环体末尾的三条增量指令：

```asm
addl $1, -12(%ebp)   ; ① -12 加 1
addl $1, -16(%ebp)   ; ② -16 加 1
addl $2, -12(%ebp)   ; ③ -12 加 2
```

**指令③ `addl $2, -12(%ebp)` 是唯一一条 +2 的指令。** for 增量中只有 `j+=2` 涉及 +2，所以：

> **`-12(%ebp)` = j**（因为它被加了 2）

一旦确定 `-12 = j`，剩下的 `-16` 就是 **i**。

#### 三条增量指令分别对应什么？

| 指令 | 对应 C 代码 | 说明 |
|:----:|-------------|------|
| ① `addl $1, -12` | `j++`（表达式 (5) 的副作用） | j 先被读取参与乘法，然后自增 |
| ② `addl $1, -16` | `i++`（for 增量） | 同时也恢复了下面 `i--` 的临时减 1 |
| ③ `addl $2, -12` | `j+=2`（for 增量） | 每轮 j 额外加 2 |

#### 为什么指令①是表达式副作用？

看 sum 的累加过程：

```asm
subl $1, -16(%ebp)        ; i--（这个 -- 是表达式里的，见下）
movl -12(%ebp), %eax      ; eax = j（读取 j 的旧值）
imull -16(%ebp), %eax     ; eax = j × (i-1)
addl -4(%ebp), %eax       ; eax = j×(i-1) + flag
addl %eax, -8(%ebp)       ; sum += 上述结果
addl $1, -12(%ebp)        ; j++（表达式里 j++ 的副作用在这里发生）
```

`movl -12(%ebp)` **先**读取了 j 的旧值参与乘法，**然后** `addl $1, -12(%ebp)` 才给 j 加 1。这正是 C 语言 **后置自增 `j++`** 的语义：**先用旧值，再自增**。

所以表达式 (5) 可以写成 `j++ * (i - 1) + flag`。

#### 最终确认

综合以上：

| 偏移 | 变量 | 初值 | 依据 |
|:----:|:----:|:----:|------|
| `-4(%ebp)` | flag | — | 唯一剩余，只被 `movl %eax, -4(%ebp)` 写入 |
| `-8(%ebp)` | sum | 17 | 唯一被立即数初始化的偏移 |
| `-12(%ebp)` | **j** | 1 | 被 `addl $2` 操作（对应 `j+=2`） |
| `-16(%ebp)` | **i** | 5 | 被 `andl $1` 检查奇偶性；被 `subl $1` 临时减 1 |

> **注意：** 初值分配是 `i=5, j=1`，而不是 `i=1, j=5`。因为 `movl $5, -16(%ebp)` 给 i 赋了 5，`movl $1, -12(%ebp)` 给 j 赋了 1。

### 总结：确定顺序的方法论

| 步骤 | 方法 | 结论 |
|:----:|------|:----:|
| ① | 找唯一被立即数初始化的偏移 | `-8` = sum |
| ② | 找被加法组合后用于条件判断的两个偏移 | `-12`, `-16` ∈ {i, j} |
| ③ | 剩余偏移 = flag | `-4` = flag |
| ④ | 找哪条 `addl $2`（只有 `j+=2` 涉及 +2） | `-12` = j, `-16` = i |

其核心思路是：**不靠猜测编译器对变量的分配顺序（不同编译器可能不同），而是通过每条指令的"语义角色"反向推导——谁被 +2（`j+=2`）、谁被检查奇偶、谁被减 1 后参与乘法。**
