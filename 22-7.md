# 32位Intel二级页表地址转换计算题

## 题目题干（20分）

某32位的Intel系统，使用2级页表处理虚拟内存地址转换，页表大小为4K，假设所有进程在特权模式下运行。内存片段内容如下表，其中所有数字都是十六进制。任何未显示的内存都假定为零。所有的页都是4K对齐的，页目录的基址（PDBA）是：`0x0045d000`。

对于以下给出的3个虚拟地址，请执行虚拟地址到物理地址的转换。如果在地址转换过程中发生错误，请找到导致失败的页表条目的物理地址。如果一级页表PDE中的有效位(P)设置为零，那么二级页表PTE地址填写（不在内存）。

### 提示

1. 本32位系统中，虚拟地址和物理地址都是32位；
2. 每个页表条目4个字节；
3. 32位页条目结构：高20位表示页表基址，低12位为权限位，最低位为有效位`P`。

## 待转换的虚拟地址

### 虚拟地址 1：0x9fd28c10

**A1）** 虚拟地址的 TLBT 为： **0x27F4A** ，TLB 命中情况：**不命中**

**A2）** TLB 不命中，跳过

**A3）** PDE 的物理地址：**0x0045D9FC** ，PTE 的物理地址：**0x0DF2A4A0**

**A4）** 地址转换情况：**失败**

- 导致失败的页表条目的物理地址为：**0x0DF2A4A0**（PTE 的 P=0，缺页异常）

---

#### 详细推导过程

**1. 拆分虚拟地址**（32位，10-10-12结构）

```
VA = 0x9fd28c10
二进制: 1001 1111 1101 0010 1000 1100 0001 0000

31~22 (PDE索引): 1001 1111 11 = 0x27F
21~12 (PTE索引): 01 0010 1000 = 0x128
11~0  (页内偏移): 1100 0001 0000 = 0xC10
```

**2. 查 TLB**

VPN = 高20位 = `0x9FD28`

TLB 为 4 组 × 2 路（8 条目），Index 位宽 = $\log_2 4 = 2$ 位：

- **TLBI** = VPN 低 2 位 = `0x9FD28 & 0x3` = `0`
- **TLBT** = VPN 高 18 位 = `0x9FD28 >> 2` = `0x27F4A`

查表 7.1 Index=0 组：

| Tag | Valid | 匹配？ |
|------|:---:|:---:|
| `0x03506` | 1 | ✗（Tag不匹配） |
| `0x27F4A` | **0** | ✗（Tag匹配但Valid=0，视为miss） |

➡️ **TLB 不命中**，需走页表遍历。

**3. 查 PDE**

$$\text{PDE物理地址} = \text{PDBA} + \text{PDE索引} \times 4$$

$$= 0x0045D000 + 0x27F \times 4 = 0x0045D000 + 0x9FC = 0x0045D9FC$$

查表 7.2：`[0x0045D9FC]` = `0x0DF2A237`

解析 PDE 条目（32位）：

```
0x0DF2A237 = 0000 1101 1111 0010 1010 0010 0011 0111

高20位 (PPN): 0000 1101 1111 0010 1010 = 0x0DF2A
→ 页表基址 = 0x0DF2A << 12 = 0x0DF2A000

低12位: 0x237 = 0010 0011 0111
bit 0 (P): 1 ✓  PDE有效，页表在内存中
```

**4. 查 PTE**

$$\text{PTE物理地址} = \text{页表基址} + \text{PTE索引} \times 4$$

$$= 0x0DF2A000 + 0x128 \times 4 = 0x0DF2A000 + 0x4A0 = 0x0DF2A4A0$$

查表 7.2：`[0x0DF2A4A0]` = `0x00324236`

解析 PTE 条目（32位）：

```
0x00324236 = 0000 0000 0011 0010 0100 0010 0011 0110

低12位: 0x236 = 0010 0011 0110
bit 0 (P): 0 ✗  PTE无效，缺页异常！
```

**5. 结论**

转换失败，失败原因是 PTE 条目的有效位 P=0，目标物理页不在内存中。导致失败的页表条目物理地址为 **`0x0DF2A4A0`**。
